How the flow works

Company is created in companies first (or already exists)
Lead is created with company_id → companies and contact_id → contacts
Lead moves through pipeline stages: new → qualified → proposed → negotiating
On sign: new row added to clients, then leads.converted_to_client_id set to that new client row
Lead record stays forever — history is preserved. Query it any time from the client side
Company name always comes from companies.company_name — one source of truth

What needs cleaning

leads.name is redundant — company name lives in companies.company_name, deal name isn't being tracked separately
No FK needed from clients→leads — the reverse link (leads.converted_to_client_id) already makes the join work both ways
contacts table holds the person; companies holds the org — both are shared between leads and clients, no duplication
Consider adding a deal_name column to leads if you want to track the opportunity name separately from the company name (e.g. "TheBoldMD — Google Ads Expansion")
